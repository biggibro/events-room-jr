# 🎲 Events Room Jackaroo — платформа для игровых офлайн-встреч для игры Jackaroo

> Веб-платформа для организации офлайн-мероприятий по настольной игре Jackaroo, записи участников, чатов событий и личной статистики.

## 📋 Описание проекта

### Идея

В городе еженедельно проводится настольная игра Jackaroo. Игрокам в Jackaroo неудобно договариваться о встречах в чатах: сложно понять, кто точно придёт, где проходит игра, есть ли свободные места и кто победил. **Events Room Jackaroo** решает эту проблему — это единая платформа, где можно посмотреть расписание игр, бесплатно записаться на официальное событие, общаться в чате события и вести личную статистику.

В проекте есть один тип событий:

- **Официальные игровые дни** — публичные события, создаются только администраторами, участие бесплатное

### Целевая аудитория

- **Игроки** — хотят находить встречи, записываться на игры, общаться с участниками и видеть свою статистику
- **Администраторы** — управляют официальными игровыми днями, локациями и пользователями

### 🖥 Ключевые экраны и функциональность

#### Для гостя (не авторизован)

| **Расписание событий** | Смотреть список официальных событий и фильтровать по дате, статусу |
| **Авторизация** | Зарегистрироваться или войти |

#### Для игрока (роль: `user`)

| Экран | Что можно делать |
|-------|-----------------|
| **Главная / лендинг** | Посмотреть описание игры Jackaroo, правила, преимущества платформы, перейти к регистрации или входу |
| **Авторизация** | Зарегистрироваться по email, войти по email / паролю |
| **Расписание событий** | Смотреть список официальных событий, фильтровать по дате, статусу; записаться с карточки |
| **Страница события** | Смотреть описание, дату, время, локацию, список участников |
| **Запись на событие** | Присоединиться к событию одному или с гостями (имена гостей) |
| **Чат события** | Читать историю переписки и получать новые сообщения в real-time через Socket.io |
| **Личный кабинет** | Редактировать профиль, видеть историю игр и официальные победы |
| **Правила игры** | Читать правила Jackaroo и справочную информацию |

#### Для администратора (роль: `admin`, `superadmin`)

| Экран | Что можно делать |
|-------|-----------------|
| **Admin-панель** | Смотреть пользователей, события и агрегированную статистику |
| **Страница локации** | Создать, редактировать и архивировать локации для проведения встреч |

| **Управление событиями** | Редактировать официальные игры, переводить между статусами, управлять составом участников |
| **Итоги игры** | Отмечать и изменять победителей в любой момент |
| **Архивация события** | Перевести событие в архив без физического удаления истории и чата |

#### Для superadmin (дополнительно к `admin`)

| Экран | Что можно делать |
|-------|-----------------|
| **Управление ролями** | Назначать пользователям роли `admin` или `user` |

### ✨ Уникальные фичи

1. **Официальные игровые дни** — публичные события создаёт только администратор, участие бесплатное
2. **Real-time чат комнаты** — история доступна через REST, новые сообщения приходят через Socket.io; чат виден только записавшимся участникам
3. **Гибкая рассадка** — `random` назначает места через платформу, `free` означает свободную посадку офлайн
4. **Личная статистика** — история игр и официальные победы в профиле игрока
5. **Запись с гостями** — участник может указать имена гостей при записи на событие
6. **Защита исхода игры** — отмечать и изменять победы участников может только администратор

## 🔌 API и real-time

### Основные REST endpoints

| Группа | Метод | Путь | Назначение |
|--------|-------|------|------------|
| Auth | POST | `/api/auth/register` | Регистрация |
| Auth | POST | `/api/auth/login` | Вход по email / паролю |
| Auth | POST | `/api/auth/refresh` | Обновление access-токена |
| User | GET | `/api/me` | Профиль текущего пользователя |
| User | GET | `/api/me/stats` | Личная статистика, прошедшие и предстоящие игры |
| User | PATCH | `/api/me` | Обновить профиль |
| Events | GET | `/api/events` | Список публичных событий с фильтрами |
| Events | GET | `/api/events/:id` | Детали события |
| Events | POST | `/api/events` | Создать официальное событие (только `admin`) |
| Events | PATCH | `/api/events/:id` | Редактировать событие (только `admin`) |
| Events | PATCH | `/api/events/:id/status` | Сменить статус события (только `admin`) |
| Events | PATCH | `/api/events/:id/winners` | Отметить или изменить победителей (только `admin`) |
| Locations | GET | `/api/locations` | Список локаций (только `admin`) |
| Locations | POST | `/api/locations` | Создать локацию (только `admin`) |
| Participation | POST | `/api/events/:id/join` | Записаться или обновить список гостей |
| Participation | DELETE | `/api/events/:id/join` | Отменить запись |
| Chat | GET | `/api/events/:id/messages` | Получить историю сообщений |
| Admin | GET | `/api/admin/users` | Список пользователей |
| Admin | PATCH | `/api/admin/users/:id/role` | Назначить роль (только `superadmin`) |

### Socket.io

```
Путь подключения: /socket.io/
Аутентификация: handshake.auth.token = JWT access token

1. Клиент подключается с JWT.
2. После connect отправляет joinEvent { eventId }.
3. Сервер проверяет, что пользователь записан на событие.
4. Socket добавляется в room event:{uuid}.
5. Клиент отправляет sendMessage { eventId, message }.
6. Сервер сохраняет сообщение и рассылает newMessage участникам комнаты.
```

## 👤 User Stories

```
Как игрок, я хочу видеть расписание официальных игровых дней,
  чтобы выбрать удобную игру и заранее записаться.

Как игрок, я хочу читать чат события и получать новые сообщения сразу,
  чтобы быть в курсе договорённостей по встрече.

Как игрок, я хочу видеть свою статистику игр и побед,
  чтобы понимать прогресс и результаты.

  чтобы управлять расписанием платформы.

Как администратор, я хочу отмечать и изменять победителей в любой момент,
  чтобы официальные результаты всегда можно было скорректировать.

Как администратор, я хочу видеть пользователей и события в админ-панели,
  чтобы контролировать работу платформы и помогать пользователям.
```

### Переменные окружения (backend)

```env
DATABASE_URL=postgresql://events:events@localhost:5432/events_room_jackaroo
DATABASE_URL_TEST=postgresql://events:events@localhost:5432/events_room_jackaroo_test
JWT_SECRET=your_access_secret
JWT_REFRESH_SECRET=your_refresh_secret
SUPERADMIN_EMAIL=owner@example.com
FRONTEND_URL=http://localhost:5173
PORT=4000
```

Роль `superadmin` назначается пользователю с email = `SUPERADMIN_EMAIL` при регистрации или логине. `superadmin` имеет все права `admin` и может назначать другим пользователям роли `admin` или `user`.
