# 🎲 Events Room Jackaroo — платформа для игровых офлайн-встреч для игры Jackaroo

> Веб-платформа для организации офлайн-мероприятий по настольной игре Jackaroo, записи участников, оплаты официальных игр, чатов событий, личной статистики и встроенного маркетплейса аксессуаров.

## 📋 Описание проекта

### Идея

В городе еженедельно проводится настольная игра Jackaroo. Игрокам в Jackaroo неудобно договариваться о встречах в чатах: сложно понять, кто точно придёт, где проходит игра, есть ли свободные места, оплачено ли участие и кто победил. **Events Room Jackaroo** решает эту проблему — это единая платформа, где можно посмотреть расписание игр, записаться на официальное событие, оплатить участие, создать свою комнату, общаться в чате события и вести личную статистику и приобрести все необходимые аксесуары игры для личного использования.

В проекте есть два основных типа событий:

- **Официальные игровые дни** — создаются администраторами, участие платное, оплата проходит через ЮKassa.
- **Неофициальные игровые дни** — создаются обычными пользователями, участие бесплатное, одобрение администратора не требуется.

### Целевая аудитория

- **Игроки** — хотят находить встречи, записываться на игры, общаться с участниками и видеть свою статистику, а так же хотят создавать дополнительные встречи для своего круга.
- **Администраторы** — управляют официальными игровыми днями, пользователями, платежами и спорными ситуациями.

### ✨ Уникальные фичи

1. **Два типа встреч** — официальные платные игры (`OFFICIAL`) и бесплатные неофициальные игровые дни (`COMMUNITY`)
2. **Оплата через ЮKassa** — участие в официальных событиях и заказы маркетплейса оплачиваются отдельно
3. **Real-time чат комнаты** — история доступна через REST, новые сообщения приходят через Socket.io
4. **Гибкая рассадка** — `random` назначает места через платформу, `free` означает свободную посадку офлайн
5. **Раздельная статистика** — официальные игры и встречи вне официального дня считаются отдельно
6. **Публичные и приватные комнаты** — приватные события видны только участникам, организатору и администраторам
7. **Защита исхода игры** — участники могут самоотмечать победу только до фиксации результата владельцем комнаты
8. **Email без спама** — письма уходят только по важным изменениям события и транзакционным операциям, а не по каждому сообщению чата
9. **Встроенный маркетплейс** — аксессуары продаются в том же приложении, но через отдельный тип платежа

### Socket.io

```
Путь подключения: /socket.io/

1. Клиент подключается с JWT.
2. После connect отправляет joinEvent { eventId }.
3. Сервер проверяет права пользователя на событие.
4. Socket добавляется в room event:{uuid}.
5. Сообщения и системные события отправляются только участникам комнаты.
```

Чат не дублируется на email. Email используется только для важных изменений события и транзакционных уведомлений.

## 👤 User Stories

```
Как игрок, я хочу видеть расписание официальных и неофициальных игровых дней,
  чтобы выбрать удобную игру и заранее записаться.

Как игрок, я хочу оплатить участие в официальной игре онлайн,
  чтобы не решать вопросы оплаты на месте.

Как игрок, я хочу читать чат события и получать новые сообщения сразу,
  чтобы быть в курсе договорённостей по встрече.

Как игрок, я хочу видеть свою статистику отдельно по официальным и неофициальным играм,
  чтобы понимать прогресс и результаты.

Как организатор комнаты, я хочу создать бесплатный неофициальный игровой день,
  чтобы собрать знакомых игроков вне официального дня.

Как организатор комнаты, я хочу назначить победителей после игры,
  чтобы итог нельзя было случайно перезаписать самоотметкой участников.

Как администратор, я хочу создавать официальные платные события,
  чтобы управлять главным расписанием платформы.

Как администратор, я хочу видеть платежи и события в админ-панели,
  чтобы контролировать работу платформы и помогать пользователям.
```
